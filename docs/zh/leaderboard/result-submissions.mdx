---
title: "结果提交"
description: "如何提交可复现的排行榜结果"
---

LexBench-Browser 接受社区通过 GitHub Issues 或 PR 提交结果。闭源 Agent 可以上榜，
但必须披露足够的 metadata，方便维护者审核和复跑。

如需 official result 协调或私有 artifact 交付方式，请联系
[lexbench@lexmount.com](mailto:lexbench@lexmount.com)。不要通过邮件发送 secrets；邮件仅用于
协调安全交付方式。

## 政策

- 允许闭源 Agent 上榜。
- official leaderboard entry 需要 maintainer review 和 maintainer rerun。
- 所选 split 里的每个任务都必须被计入。
- 不得提交 API key、cookies、账号凭证或未脱敏的 provider logs。
- judge/model 变更不单独开新 leaderboard family，但每个结果必须记录 judge model 和 strategy。

## 必填 metadata

使用 `community/results/example/submission.json` 作为模板。

必填字段：

- benchmark、split 和 benchmark version
- agent name、version、source status 和 reproducibility limitations
- model provider 和 model ID
- browser backend
- judge model 和 strategy
- run command 和 eval command
- success rate、average steps、average end-to-end latency
- run artifacts 或外部 artifact URL
- skips、retries、provider incidents 和 browser constraints

## 审核流程

1. 创建 Result Submission issue 或 PR。
2. 附上 `submission.json` 和脱敏后的 artifacts。
3. 运行 metadata validator。
4. 维护者进行完整或抽样复跑。
5. 结果通过后，可以合入 public leaderboard snapshot。

## 校验 metadata

```bash
uv run python scripts/validate_result_submissions.py
```

validator 会检查必填字段、任务计数、基础分数范围、静态 leaderboard JSON，以及明显像 secret 的字段。

## 示例

```bash
cp community/results/example/submission.json /tmp/submission.json
```

填写 metadata，并链接 run directory：

```text
experiments/LexBench-Browser/All/<agent>/<model-id>/<timestamp>/
```

完整政策见仓库根目录 `EVALUATION_PROTOCOL.md`。
