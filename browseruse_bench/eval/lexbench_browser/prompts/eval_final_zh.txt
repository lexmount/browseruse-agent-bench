## 重要原则 —— 网站现状例外（Site-state Exception）

参考答案中的步骤、选项、筛选条件均基于数据集创建时的网站快照，可能已与当前
真实站点存在差异。评分时若 Agent 同时满足以下两个条件，应基于"是否做出了合理
判断并诚实告知用户"打分，而不是机械对比是否字面满足参考答案：

1. 最终截图中可清晰看出某个 reference 要求的选项（按钮 / 筛选项 / 分类 / 排序 /
   品牌 / 颜色 / 价格区间 / 字段等）在当前实际站点上已不存在、被改名，或可选
   集合发生变化。
2. Agent 在 final answer 中明确告知用户该限制（可附带也可不附带合理替代方案
   —— 替代方案是加分项，不是必要条件）。

满足两条时，对应评分项按以下比例给分：
- 完整识别站点限制 + 诚实告知（无论是否提供替代） → 该项 80–100% 满分
- 既未识别也未告知（agent 凭空完成了一个不存在的操作，或捏造站点上不存在的
  数据）→ 仍按 0 分扣

如果 Agent 仅口头声称"选项不存在"但最终截图无法佐证，视为未完成，不适用本例外。

注意：本例外只针对"reference 要求的选项在站点上已变化"这种情况，不应被泛化
为"对答案与截图证据的整体一致性"打折扣。其它评分项的标准维持不变。
